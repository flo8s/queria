# metadata.json Schema Specification

metadata.json is the dataset metadata file.
It is generated by the `queria metadata` command from `dataset.yml` and dbt's `manifest.json` / `catalog.json`.

## Structure

```
metadata.json
├── title          Catalog name
├── description    Catalog description
├── tags[]         Classification tags
├── ducklake_url   DuckLake endpoint URL
├── schemas
│   └── {schema_name}
│       ├── title      Schema name
│       └── tables[]
│           ├── name           Table identifier (dbt model name)
│           ├── title          Table display name
│           ├── description    Table description
│           ├── tags[]         Classification tags
│           ├── license        License
│           ├── license_url    License URL
│           ├── source_url     Data source URL
│           ├── materialized   Materialization type (table/view/incremental)
│           ├── sql            Compiled SQL (optional)
│           └── columns[]
│               ├── name        Column name
│               ├── description Column description
│               └── data_type   Data type
├── lineage
│   ├── parent_map   DAG adjacency list (child → parents)
│   └── nodes        Node metadata keyed by short name
└── dependencies[]   Cross-dataset dependencies (optional)
    ├── alias          Alias for the dependency
    └── ducklake_url   DuckLake URL of the dependency
```

## Field Definitions

### Top Level

| Field | Type | Required | Description |
|---|---|---|---|
| title | string | Yes | Display name of the catalog |
| description | string | Yes | Description of the catalog |
| tags | string[] | No | Classification tags |
| ducklake_url | string | Yes | DuckLake endpoint URL |
| schemas | object | Yes | Object keyed by schema name |
| lineage | object | Yes | DAG and node metadata |
| dependencies | object[] | No | Cross-dataset dependencies |

### schemas.{name}

| Field | Type | Required | Description |
|---|---|---|---|
| title | string | Yes | Display name of the schema |
| tables | object[] | Yes | Array of tables |

### schemas.{name}.tables[]

| Field | Type | Required | Description |
|---|---|---|---|
| name | string | Yes | Table identifier (matches dbt model name) |
| title | string | Yes | Display name of the table |
| description | string | Yes | Description of the table |
| tags | string[] | No | Classification tags |
| license | string | No | License (e.g., "CC BY 4.0") |
| license_url | string | No | License URL |
| source_url | string | No | Data source URL |
| materialized | string | Yes | Materialization type (table/view/incremental) |
| sql | string | No | Compiled SQL |
| columns | object[] | Yes | Array of columns |

### columns[]

| Field | Type | Required | Description |
|---|---|---|---|
| name | string | Yes | Column name |
| description | string | Yes | Description of the column |
| data_type | string | Yes | Data type (from dbt catalog.json, falls back to manifest.json) |

### lineage

| Field | Type | Required | Description |
|---|---|---|---|
| parent_map | object | Yes | DAG adjacency list: `{child_name: [parent_names]}` |
| nodes | object | Yes | Node metadata keyed by short model name |

### lineage.nodes.{name}

| Field | Type | Required | Description |
|---|---|---|---|
| fqn | string[] | Yes | Fully qualified name |
| resource_type | string | Yes | Resource type (e.g., "model") |
| config | object | Yes | Node config (`{materialized: "table"}`) |
| meta | object | Yes | Node meta properties |

### dependencies[]

| Field | Type | Required | Description |
|---|---|---|---|
| alias | string | Yes | Alias for the dependency |
| ducklake_url | string | Yes | DuckLake URL of the dependency |

## Data Sources

- Top level / schemas: retrieved from `dataset.yml`
- Tables / columns: retrieved from dbt's `manifest.json` (each model's `meta` and `columns`)
- Column data_type: retrieved from dbt's `catalog.json`, falls back to `manifest.json` (available with contract-enforced models)
- Lineage: extracted from dbt's `manifest.json` (`parent_map` and `nodes`)
- Public model filtering: `meta.public: true` in dbt model YAML

## Design Decisions

1 datasource = 1 DuckLake (see multi-datasource.md for details):
- 1 metadata.json = 1 datasource → no `databases` wrapper needed, placed directly at the top level
- Multi-language support deferred → plain strings instead of `{ja: ..., en: ...}`
- `display_name` → unified to `title` for easier mapping to DCAT's `dct:title`
- Eliminated `meta` nesting → `title`, `tags`, `license`, etc. placed flat on each table
- Single file structure → no need to split while the data volume is small

## Future Extension Paths

- Multi-language: extend `title` to `title: {ja: "...", en: "..."}`
- DCAT conversion: add a script to convert metadata.json → DCAT (JSON-LD)
- File splitting: split by schema when the number of tables grows
- CMS integration: generate metadata.json from a CMS (headless CMS, etc.)

## Adding a New Field

1. Add the new field to `dataset.yml` or dbt model's `meta`
2. Add the field to the Pydantic model in `src/queria/models.py`
3. Process the field in the generation logic in `src/queria/run.py`
4. Update the field definition tables in this document
5. Verify generation with `uv run queria metadata <datasource>`
